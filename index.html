What it is

a gesture-driven
robot arm monitor

This interface reads the tracking feed generated by the vision pipeline, reshapes it into browser-ready telemetry, and animates the arm state in real time. The page is not a mockup anymore: it is driven by the same hand pose and joint data the controller uses.

Hand coordinates, motion energy, and arm pose are normalized by Flask and exposed as a low-latency stream. These cards stay in sync with the animated canvases below.

The pipeline

From camera frame
to arm pose

The tracker writes snapshots to disk, Flask watches the feed, and the browser subscribes over server-sent events. The result is a lightweight local stack that is simple to run and smooth enough for live visual feedback.

Shoulder, elbow, wrist, and gripper state are rendered directly from the streamed joint values. When the real feed is unavailable, the server supplies a demo trace so the page still animates and remains testable.
